# IsoChrome (WPF)

**IsoChrome** — портативное WPF-приложение для управления изолированными сессиями **Google Chrome**: быстрые временные сессии, постоянные профили, закладки, история запусков.
 
> Текущая версия: **0.4.108**

## Для чего это

IsoChrome помогает держать разные “контексты” Chrome раздельно:

- временные сессии для разовых задач (без смешивания куки/кэша);
- постоянные профили для проектов/клиентов/аккаунтов;
- быстрый запуск набора вкладок по шаблону;
- единое место для закладок и быстрого открытия.

## Для кого подойдёт

### Если тебе нужно разделять аккаунты и рабочие контексты

- работа с несколькими аккаунтами на одних и тех же сервисах (несколько кабинетов/логинов);
- разные проекты/клиенты с отдельными cookie/кэшем/настройками;
- тестирование входа/регистраций/ролей без “смешивания” данных;
- параллельная работа в разных окнах Chrome, когда важно, чтобы они не влияли друг на друга.

### Если тебе важны быстрые запуски «под задачу»

- QA/тестировщикам и разработчикам — быстро поднять чистую сессию, проверить сценарий и закрыть;
- менеджерам/аналитикам/маркетологам — быстрый старт нужных вкладок через шаблоны;
- саппорту — открыть сайт в изолированной сессии и воспроизвести проблему пользователя;
- фрилансерам — держать “разные рабочие среды” для разных заказчиков.

### Если тебе нужно простое локальное управление закладками

- группировка/теги/заметки;
- импорт/экспорт;
- дедупликация по URL;
- проверка доступности ссылок.

## Кому может не подойти

- если тебе нужен полноценный «антидетект браузер» с подменой отпечатков (fingerprint) — IsoChrome не про это;
- если ты используешь не Google Chrome (например, только Edge/Firefox) — приложение запускает именно Chrome;
- если нужна синхронизация профилей между устройствами — здесь хранение локальное, portable.

## Возможности

### Быстрая сессия

- запуск отдельного экземпляра Chrome с временным профилем;
- режимы: обычный / инкогнито;
- User-Agent: по умолчанию / случайный / свой;
- список активных сессий:
  - открыть URL в выбранной сессии;
  - закрыть выбранную/все сессии.

### Профили

- создание и запуск постоянных профилей;
- для профиля можно настроить:
  - инкогнито;
  - отключение фонового режима;
  - пользовательский User-Agent;
  - стартовые URL (включая список URL из закладок);
  - восстановление вкладок при следующем запуске;
- управление профилями:
  - переименование (с переименованием папки);
  - закрепление (★) и сортировка закреплённых выше;
  - открыть папку профиля;
  - очистка данных/удаление;
  - принудительное завершение запущенного профиля.

### Закладки

- основное хранилище `bookmarks.json` + синхронизация в `bookmarks.txt` для совместимости;
- поля: название, URL, группа, теги, заметка, закрепление;
- импорт/экспорт TXT/CSV/JSON;
- фильтр по группе + поиск по всем полям;
- дедупликация по URL;
- проверка доступности (HTTP) с отображением статуса;
- шаблоны запуска (наборы URL) + управление шаблонами.

### Недавние

- журнал запусков и быстрый повторный запуск.

## Требования

- Windows 10/11
- Установленный Google Chrome

## Установка и запуск (portable)

1. Скачай последнюю версию из раздела релизов.
2. Распакуй архив в любую папку (например, `D:\Apps\IsoChrome`).
3. Запусти `IsoChrome.exe`.

## Где хранятся данные

### Основная база данных приложения

По умолчанию **все пользовательские данные лежат рядом с `IsoChrome.exe`** (портативный режим):

- `profiles\` — постоянные профили.
- `profiles\_quick\` — служебные файлы быстрых сессий (настройки/журнал/шаблоны).
- `bookmarks.json` — основное хранилище закладок.
- `bookmarks.txt` — текстовая копия закладок для совместимости.
- `crash.log` — лог критических ошибок (если случился краш).

### Миграция со старых версий

Если в `%AppData%\IsoChrome` есть данные и в текущей папке приложения данных ещё нет, IsoChrome попробует аккуратно перенести:

- `bookmarks.txt`, `bookmarks.json`, `bookmarks.json.bak`
- папку `profiles\`

### Временные профили быстрых сессий

Временные профили быстрых сессий создаются отдельно и удаляются после завершения сессии:

- обычно: `%ProgramData%\IsoChrome\quick_profiles\`
- fallback (если нет прав): `%TEMP%\isochrome_temp_...`

## FAQ / Решение проблем

### Chrome не найден

Убедись, что Google Chrome установлен в стандартный путь (Program Files / Program Files (x86) / LocalAppData).

### Приложение не запускается / белое окно

Проверь файл `crash.log` рядом с `IsoChrome.exe` и приложи его в обращении.

### Где мои профили/закладки

Смотри раздел **«Где хранятся данные»** — по умолчанию всё лежит рядом с `IsoChrome.exe`.

## Приватность и безопасность

- IsoChrome не является браузером и не проксирует трафик — он **запускает Chrome** с параметрами профиля.
- Все данные профилей/куки/кэш хранятся в файловой системе (в папках профилей Chrome).
- Закладки и настройки приложения хранятся локально в файлах рядом с приложением.
